### Splash mechanism ‚ÄĒ HTML in `index.html`

The cleanest pattern: include the splash markup directly inside `<div id="root">` in `index.html`. Browser paints it on first frame (before any JS runs). When `main.tsx` calls `createRoot(root).render(<App />)`, React replaces the `#root` children with the real shell. The splash is automatically gone ‚ÄĒ no timer-based "hide" logic.

```html
<div id="root">
  <div class="boot-splash">
    <img src="/monogram.png" alt="4neverCompany OS" class="boot-splash-logo" />
    <p class="boot-splash-credit">Powered by Paperclip ¬∑ Hermes Agent ¬∑ BMAD Method ¬∑ Tauri ¬∑ Zellij ¬∑ Claude Code</p>
  </div>
</div>
```

The `.boot-splash` CSS lives inline in `<style>` inside `<head>` so the paint doesn't wait for a stylesheet GET. Once React mounts (~200‚Äď800ms depending on cold start), the splash vanishes naturally ‚ÄĒ comfortably under the AC's 2-second cap.

The short credit string is constructed at build time from `renderAttributionShort(BUNDLED)` and injected into `index.html` via a Vite plugin OR ‚ÄĒ simpler ‚ÄĒ hand-maintained to match the source. **Recommend: hand-maintained.** The set of bundled-upstream names changes infrequently (when we add or remove a dependency) and the tests guard against drift between the canonical source and what the splash shows.

## Risks

- **Drift between `attribution.ts` and `pinned-versions.md`** ‚ÄĒ when a Tier-1 component bumps in `pinned-versions.md`, someone has to update `versions.ts`. Mitigation: `attribution.test.ts` includes a guardrail that fails if any `BUNDLED` entry is missing from `VERSIONS`. Doesn't catch stale versions (no automated diff against `pinned-versions.md`); flag as "discipline note" in `versions.ts` module-doc + a `references/reference-bmad-method-workflow.md` addition.
- **DoneStep visual regression.** The wizard's `DoneStep.tsx` currently has a compact, centered help-copy line. Replacing it with a full attribution block changes the visual weight of the card. Mitigation: keep the card compact ‚ÄĒ show the top-3 bundled upstreams inline + "See full credits in Settings ‚Üí About ‚Üí" CTA. The full block lives in Settings ‚Üí About.
- **Splash flash-of-unstyled-content.** Browser paints `#root`'s HTML splash on first frame, then React mounts and replaces it. If the React shell takes > 2s to render, the splash exceeds the AC's "‚Č§ 2 seconds" cap. Mitigation: monitor React mount time during visual smoke; if slow, code-split the shell. Should not be a concern at 1.19 scope.
- **`tauri-plugin-opener` scope.** The "Read full LICENSES.md ‚Üí" button needs to open the LICENSES.md file. If the opener plugin's scope (in `tauri.conf.json`'s permissions) doesn't allow opening repo-rooted files, the button silently fails. Mitigation: verify scope in the dev story; if needed, expand `dialog:allow-open` analog or add `opener:allow-open-url` for `file://` scheme.
- **Settings rail item conflicts with future polish.** Story 1.19 owns the entire `settings` rail item slot. Future stories that add Personas / Vault / Theme sub-panels under Settings will edit `SettingsView.tsx`; this story should leave a clear "TODO(M2): Personas/Vault sub-panels" comment so the extensibility story isn't surprising.
