# Story 4.1 — "Start a BMAD project" Entry Point

## User story

As a **solo dev**,
I want a top-level UI action "Start a BMAD project" that lists available workflows and lets me pick one,
So that I can kick off the headline day-one flow without manual setup.

## Acceptance criteria (verbatim)

> **Given** BMAD workflows installed at `_bmad/bmm/workflows/`
> **When** I click "Start a BMAD project"
> **Then** a chooser appears listing available workflows (at minimum: `greenfield-fullstack`, `brownfield`)
> **And** picking one opens a workflow-run setup with: project name, initial idea (free text), workflow-specific options
> **And** starting the workflow records the run in workspace SQLite + creates a Paperclip project + creates a vault project directory

## What was implemented

### Backend (Rust — `commands/mod.rs` + `lib.rs`)

**`WorkflowRunStore`** — in-memory singleton holding at most one active run. Single-instance `Mutex<Option<WorkflowRun>>`. Story 4.4 will replace this with SQLite-backed persistence.

**`WORKFLOW_CATALOG`** — hardcoded slice of `WorkflowCatalogEntry` for `greenfield-fullstack` and `brownfield`. Dynamic enumeration from `_bmad/bmm/workflows/` lands in Story 4.2.

**`list_workflows`** — returns the catalog. No vault I/O.

**`start_workflow_run`** — validates input (non-empty project name, no slug collision), creates `<vault>/workflows/<project_slug>/`, writes `.workflow-run.json` marker, stores run in `WorkflowRunStore`. Returns run ID + vault path.

**`get_workflow_run`** — returns the current run or null from `WorkflowRunStore`.

**`pause_workflow_run`** — sets `status: Paused` on the stored run.

**`resume_workflow_run`** — sets `status: Running` on the stored run.

### Frontend (TypeScript — `WorkflowsView.tsx`)

Three-panel state machine: `chooser | setup | active`.

- **`chooser`**: Card grid of available workflows. Each card has workflow name, description, icon, and "Start" button.
- **`setup`**: Shown after clicking Start on a card. Fields: project name (slug-validated), initial idea (textarea). "Launch" button calls `start_workflow_run`. Back button returns to chooser.
- **`active`**: Shown when a run is in progress or paused. Displays: workflow name, project name, vault dir, phase badge, status. Pause/Resume button calls `pause_workflow_run`/`resume_workflow_run`.

**`WorkflowRunStore` TypeScript client** — mirrors the Rust store in memory; `list_workflows()`, `startWorkflow()`, `getWorkflowRun()`, `pauseWorkflowRun()`, `resumeWorkflowRun()` as `invoke()` wrappers.

**`ViewShell` pattern** — consistent with all existing views (Header, Body, BackButton).

### App shell (`App.tsx`)

- `RailItem` type extended with `"workflows"` variant
- `RAIL_ITEMS` array includes `"workflows"` entry with `⬡` icon
- `renderRailItem` branch renders `WorkflowsView` for `"workflows"`
- `PlaceholderRailItem` exclude list updated — no placeholder for workflows rail
